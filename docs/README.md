# Docs

This folder is the source of truth for how we design evals and run benchmarks in this repository.

If you are starting fresh, read these in order:

1. [`adding-new-eval.md`](./adding-new-eval.md) for the eval file contract and `requirements.yaml` format.
2. [`starter-scaffold-contract.md`](./starter-scaffold-contract.md) for starter `app/` baseline policy.
3. [`testing-your-evals.md`](./testing-your-evals.md) for local validation workflow before opening a PR.
4. [`benchmarking-selected-models.md`](./benchmarking-selected-models.md) for model comparison runs and interpretation.
5. [`adding-new-category.md`](./adding-new-category.md) for category-level research workflow and traceability expectations.

For contribution workflow and PR conventions, see [`../AGENTS.md`](../AGENTS.md).
